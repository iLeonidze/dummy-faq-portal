Basics

What is nginx typically used for?

nginx is commonly used as a web server, reverse proxy, load balancer, and HTTP cache. In practice it often sits in front of an application, terminates client connections, and serves static assets directly.

Static hosting

Can nginx host a static site by itself?

Yes. A static HTML site is one of the simplest nginx workloads. This this portal follows that model: static pages, shared CSS and JS, and no application runtime behind them.

Configuration

Where does nginx configuration usually live?

The main file is often called nginx.conf, commonly under /etc/nginx/. Larger setups include additional config snippets from dedicated directories so server blocks stay readable.

Validation

How should I validate changes before a reload?

Run nginx -t first. Once syntax and includes pass, reload the master process so the updated config is applied without the disruption of a full stop/start cycle.

Proxying

Why put nginx in front of an app?

It can centralize TLS termination, serve static files, normalize request handling, and forward dynamic traffic to upstream services. That gives the application fewer edge-facing concerns to manage.

Troubleshooting

Where do I start when something returns 404 or 502?

For 404s, check whether the request path maps to a real file or location rule. For 502s, inspect upstream health, ports, timeouts, and the error log to see whether nginx could reach the backend.
